# 周杰伦粉丝知识答题（Web 先行版）

Nuxt 4 + TypeScript + Nitro 实现的移动端答题应用，面向周杰伦粉丝群体，具备双模式答题、粉丝等级、周榜名人榜、账号体系与本地战绩。架构完整对齐微信小程序迁移。

## 快速开始

```bash
npm install
npm run dev     # http://localhost:3000
npm run build   # 生产构建
```

## 核心功能（2025 最新）

### 🎖️ 粉丝等级系统
- 基于**答题准确率加权计算**，按题型难度分配权重：
  - **填空 40%**（最难）
  - **单选 35%**（中等）
  - **判断 25%**（最简单）
- 统计范围：**最近 50 道答题**，满分 100 分
- 等级划分：
  - 0-40 分：路人粉
  - 41-70 分：普通粉
  - 71-90 分：资深粉
  - 91-100 分：骨灰铁粉
- 答题不足 50 道时显示**锁定状态**，点击徽章直接进入答题解锁

### 🎮 双模式答题

#### 休闲模式（quiz.vue）
- 支持**选择题库分类**：混合 / 歌词 / 创作背景 / 个人经历
- 答题进度**自动持久化**：中途退出下次进入自动从断点续答
- 分类进度隔离：不同题库的进度互不干扰
- **不记录战绩**，只更新本周最高分
- 结算弹窗二选一：参与排行榜（需登录）or 仅保存本地
- 登录后支持**自动上传本地最高分**

#### 冒险模式（adventure.vue）
- 固定**混合题库**，挑战极限
- 所有答题**永久记录到「冒险战绩」**，支持统计正确率
- 成绩计入「本周名人榜」排名

### 🏆 本周名人榜（原周榜排行榜）
- 后端驱动排序，规则：**分数降序 → 用时升序 → 提交时间**
- 已登录用户答题后自动上传本周最高分
- 每周一 00:00 自动清零，与本地最高分周期完全同步

### 📝 我的战绩
- **仅记录冒险模式**成绩，休闲模式不统计
- 展示最高分、总场次、正确率、累计答题数
- 支持清除本地记录

### 🔐 账号体系
- 注册：账号 + 密码 + 昵称（bcrypt 加密）
- 登录：JWT Token，本地持久化
- 游客模式：不登录也能完整答题，结算时选择是否参与排名

### 📦 题库与存储
- 内置 60+ 道种子题，覆盖单选/判断/填空三种题型
- 题库优先级：KV 数据库 > 种子数据兜底，KV 为空时自动用内置题目
- 存储抽象层：支持 Vercel KV / 本地文件系统无缝切换（只需改 `nuxt.config.ts`）

## 游戏规则（首页弹窗说明）

1. **粉丝等级**：按最近 50 道答题准确率加权计算，答题越多等级越准确
2. **休闲模式**：可选择特定分类刷题，不记录战绩，适合查漏补缺
3. **冒险模式**：固定混合题库，难度均衡，所有对局永久记录并参与名人榜排名
4. **名人榜**：每周一 00:00 清零重计，同分用时短者优先
5. **游客模式**：不登录也能答题，结算时可选择是否参与排名
6. **断点续答**：休闲模式支持中途退出，下次进入继续答题

## API 接口

| Method | Path | 说明 |
| --- | --- | --- |
| POST | `/api/auth/register` | 注册 |
| POST | `/api/auth/login` | 登录，返回 Token |
| GET  | `/api/auth/me` | 当前用户信息 |
| POST | `/api/auth/logout` | 退出登录 |
| GET  | `/api/questions/random` | 随机抽题（`count`, `category`） |
| GET  | `/api/questions` | 题目列表（支持分页筛选） |
| POST | `/api/questions` | 新增/编辑题目 |
| POST | `/api/questions/delete` | 删除题目 |
| GET  | `/api/rankings` | 本周名人榜 |
| POST | `/api/rankings/submit` | 上传成绩 |

题目管理后台系统设计文档见：[`docs/admin-system.md`](docs/admin-system.md)

## 部署

- **自动部署**：`git push origin master` 自动触发 Vercel 部署（已配置）
- 前端 + 后端 一体部署到 Vercel（`vercel.json` 已配置 `NITRO_PRESET=vercel`）
- 持久化默认使用 Nitro `fs` 驱动，切换到 Vercel KV 只需修改 `nuxt.config.ts` 中 `nitro.storage.db.driver = 'vercel-kv'`
- 微信小程序迁移指南：[`docs/MIGRATION.md`](docs/MIGRATION.md)

## 技术要点

- **周周期一致性**：前后端使用同一套 `getWeekKey` 算法，确保周榜清零与本地最高分周期完全同步
- **存储抽象**：`unstorage` 统一封装，支持多驱动切换，无需修改业务代码
- **断点续答**：本地存储记录当前题目位置与分类，退出重进无缝恢复
- **粉丝等级加权**：难度越高的题目对最终等级影响越大，真实反映粉丝深度
- **渐进式登录**：答题全程无需登录，仅在需要参与排名时引导登录，降低用户门槛
