# 周杰伦粉丝知识答题周榜小程序（Web先行版）

## 项目基础信息
- **项目名称**：周杰伦粉丝知识答题周榜小程序
- **技术栈**：Nuxt3 + TypeScript（前端），Nitro（后端）
- **UI风格**：周杰伦复古国风/专辑质感，移动端优先
- **部署**：Vercel（前端）+ Vercel KV（后端数据）
- **核心目标**：Web先行开发，架构完全兼容微信小程序迁移

## 核心需求概述
开发周杰伦题库答题移动端项目，包含：歌词/创作背景/个人经历三类答题、个人战绩本地存储、后端驱动周榜排行榜、极简账号密码登录注册。全程为微信小程序迁移做架构兼容。

---

## 一、前端核心功能（Nuxt3 + TS）

### 1. 首页
- 周杰伦主题国风UI（复古黑胶/磁带风格，墨绿/酒红配色）
- 功能入口：开始答题、周榜排行榜、题库分类、我的战绩、规则说明
- 登录/注册入口（右上角）
- **权限规则**：未登录可刷题，无法上榜、无法上传分数

### 2. 核心答题页
- 支持单选/判断题/填空题
- 可选答题倒计时（开关控制）
- 答对答错展示详细知识点解析
- 答题完成自动计分、生成粉丝等级：
  - 路人粉（0-40分）
  - 普通粉（41-70分）
  - 资深粉（71-90分）
  - 骨灰铁粉（91-100分）

### 3. 周榜排行榜页面（后端驱动）
- **数据来源**：全部请求后端接口，无本地缓存
- **排序规则**：本周最高得分降序 → 答题用时 → 答题时间
- **展示字段**：排名、用户昵称、本周最高分、答题场次、粉丝等级、上榜时间
- **提交逻辑**：已登录用户答题结束自动上传最高分，覆盖本人本周最高记录
- **权限拦截**：未登录用户点击排行榜跳转到登录页

### 4. 我的战绩（本地存储）
- 个人所有历史答题记录、分数统计、刷题数据
- 本地持久化（localStorage 统一封装）
- 仅个人可见

### 5. 题库分类 + 规则说明
- 按歌词/创作背景/个人经历三类分类刷题
- 弹窗展示：计分规则、榜单周期规则、粉丝等级规则

### 6. 登录注册页面
- 极简账号密码体系
- 注册：账号 + 密码 + 昵称
- 登录：账号 + 密码
- Token 持久化存储
- 退出登录功能

## 二、后端核心功能（Nitro）

### 1. 题库托管接口
- 所有答题题库后端存储（歌词50%、创作背景30%、个人经历20%）
- 接口：
  - `GET /api/questions/random?count=10&category=xxx` - 随机获取题目
  - `GET /api/questions/categories` - 获取分类列表
- 支持分类筛选、分页、随机抽取
- 初始化种子数据（每类至少20题）

### 2. 周榜排行榜接口
- `GET /api/rankings` - 获取本周排名列表
- `POST /api/rankings/submit` - 提交答题分数（已登录用户）
- **规则**：单用户一周仅保留一条最高得分记录，新分更高才更新

### 3. 定时任务
- 每周一 00:00:00 自动清空本周榜单数据
- 单条榜单数据仅保留1周，过期自动清除

### 4. 用户登录注册模块
- `POST /api/auth/register` - 注册（账号、密码、昵称）
- `POST /api/auth/login` - 登录（账号、密码）→ 返回 Token
- `GET /api/auth/me` - 获取当前用户信息（昵称、本周最高分、答题统计）
- `POST /api/auth/logout` - 退出登录
- 密码加密存储（bcrypt）

### 5. 预留功能（逻辑完整，入口隐藏）
#### 拼图功能
- 后端：`GET /api/puzzle/images` - 获取图片资源列表
- 前端：封装页面组件骨架，功能入口隐藏

#### 听歌识曲功能
- 后端：`GET /api/music/clips` - 获取音频片段，`POST /api/music/verify` - 识曲匹配校验
- 前端：封装音频请求、播放、匹配逻辑，页面入口隐藏

## 三、架构规范（小程序迁移兼容）

### 1. 代码分层
- `/composables` - 业务逻辑（答题、排行榜、用户）
- `/utils` - 工具函数（storage、request、加密）
- `/types` - 统一 TypeScript 类型定义
- `/server/api` - 后端接口
- `/pages` - 视图层（仅UI，业务逻辑抽离到 composables）

### 2. 兼容要求
- 不使用 Web 专属 API（window、document 等，用 Nuxt 兼容层）
- 本地存储工具统一封装 `useStorage()`，预留小程序 storage 替换入口
- 请求逻辑统一封装 `useRequest()`，预留小程序 request 替换入口
- 页面路由、组件命名对齐微信小程序规范
- 样式使用 rpx/rem 单位，兼容小程序

### 3. 功能开关
- `app.config.ts` 配置开关：
  - `enablePuzzle: false`
  - `enableMusicRecognition: false`
- 后续仅修改开关即可开启预留功能
